# HarmonyOS 项目规则

## 文档语言

**所有文档必须使用中文编写**，包括但不限于：
- 功能规范（spec.md）
- 实施计划（plan.md）
- 任务列表（tasks.md）
- 检查清单（checklists/*.md）
- 代码注释
- 提交信息

## 设计规范（必须遵循）

参考华为官方设计指南：
- [设计最佳实践](https://developer.huawei.com/consumer/cn/doc/design-guides/practices-overview-0000001746498066)

### UX体验标准
遵守华为应用UX体验标准：
- [UX体验标准](https://developer.huawei.com/consumer/cn/doc/design-guides/ux-guidelines-general-0000001760708152)

## 设计文档要求

所有设计文档（plan.md、quickstart.md、tasks.md）必须：
1. 引用已有skill
2. 使用完整命令路径
3. 遵循华为设计最佳实践
4. 符合UX体验标准

## TDD 开发模式

严格遵循红-绿-重构循环：
1. 🔴 红：先编写失败的测试用例
2. 🟢 绿：编写最少代码使测试通过
3. 🔄 重构：优化代码结构

### TDD 强制要求

**所有功能开发必须遵循 TDD**，任务列表必须包含：

| 任务类型 | 说明 | 标记 |
|----------|------|------|
| 测试任务 | 先编写测试用例（预期失败） | 🔴 |
| 实现任务 | 编写最小实现代码（使测试通过） | 🟢 |
| 重构任务 | 优化代码结构和性能 | 🔄 |

**任务列表格式要求**：
- 每个用户故事阶段必须按 🔴测试 → 🟢实现 → 🔄重构 顺序组织
- 测试任务必须在实现任务之前
- 测试文件路径必须明确指定

## 任务执行规则

### 执行任何任务前必须：

1. **完整阅读 CLAUDE.md**：检查所有适用的项目规则
2. **确认开发模式**：本项目的 TDD 开发模式适用于所有功能开发
3. **确认文档语言**：所有文档使用中文
4. **确认参考规范**：必须真正阅读并集成官方文档内容

### 任务前检查清单

```
□ 已检查 CLAUDE.md 中的所有规则
□ 已确认 TDD 开发模式要求
□ 已确认文档语言要求（中文）
□ 已确认参考文档规范（必须真正阅读内容）
```

## 代码规范

- 遵循 ArkTS 语法规范
- 使用显式类型声明（禁止 any/unknown）
- API Key 必须通过用户设置界面配置，禁止硬编码

## 参考文档规范

### 必须真正阅读并集成内容

当要求参考官方文档或最佳实践时，必须：

1. **禁止只引用链接** - 必须实际获取并阅读文档内容
2. **必须输出证据** - 引用文档原文，证明已阅读
3. **必须集成到交付物** - 将内容真正写入规范/计划/代码中

### 遇到困难时的处理方式

| 困难 | 处理方式 |
|------|----------|
| 网页抓取失败 | 使用搜索获取摘要，或告知用户无法访问 |
| 内容不完整 | 多次尝试不同方式获取，不满足于部分结果 |
| 无法验证 | 主动向用户说明局限性 |

### 用户验证方式

用户可通过以下方式验证是否真正参考了文档：
- "请引用文档原文"
- "告诉我具体内容是什么"
- "列出文档目录结构"


### 经验总结
把犯的错误，错误的原因，解决方案总结记录到experience.md，便于以后避开
